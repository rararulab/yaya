## Philosophy
The kernel. Event bus, plugin ABI, closed public event catalog, and (later) the plugin registry and fixed agent loop. Every other yaya capability depends on this layer. See [GOAL.md](../../../GOAL.md) and [docs/dev/plugin-protocol.md](../../../docs/dev/plugin-protocol.md).

## External Reality
- [`docs/dev/plugin-protocol.md`](../../../docs/dev/plugin-protocol.md) is the authoritative contract тАФ this module is its Python mirror; drift fails review.
- `tests/kernel/` is ground truth for behavior; тЙе80% coverage enforced by `pyproject.toml`.
- `mypy --strict` + ruff are the import-graph enforcers: the kernel MUST NOT import from `cli`, `plugins`, or `core`.
- BDD contract: [`specs/kernel-bus-and-abi.spec`](../../../specs/kernel-bus-and-abi.spec); `agent-spec guard` rejects unbound scenarios.

## Constraints
- `events.py` тАФ closed `PublicEventKind` Literal, per-kind `TypedDict` payloads, `Event` dataclass, `new_event()` factory. New public kind = governance amendment (protocol doc + GOAL.md + this module in the same PR).
- `plugin.py` тАФ `Category` StrEnum, runtime-checkable `Plugin` Protocol, `KernelContext` (emit stamps `source` with the plugin name; plugins cannot forge identity).
- `bus.py` тАФ asyncio pub/sub. Exact-kind routing (no wildcards at 1.0). Per-subscriber 30s timeout. FIFO per `session_id` via a single drain worker task over a per-session `asyncio.Queue`; handlers may call `publish` / `ctx.emit` on the same session while running, the follow-up event is enqueued and delivered after the current handler returns (no re-entry hazard). Failing handlers produce a synthetic `plugin.error` enqueued on the `"kernel"` session (`source = "kernel"`); kernel-origin failures do NOT re-emit (recursion guard).
- `loop.py` тАФ fixed agent loop (the scheduler). `AgentLoop(bus, config)` subscribes to `user.message.received`, `user.interrupt`, and the four response kinds (`strategy.decide.response`, `llm.call.response`, `llm.call.error`, `memory.result`, `tool.call.result`); emits requests in the order frozen by [`docs/dev/plugin-protocol.md`](../../../docs/dev/plugin-protocol.md) (strategy тЖТ optional memory тЖТ llm тЖТ zero-or-more tools тЖТ `assistant.message.done` тЖТ optional `memory.write`). Strategy plugins choose the `next` step (`"llm" | "tool" | "memory" | "done"`); the loop owns the order. **Correlation** is by originating event `id`: each outbound request's `id` is mirrored back as `payload.request_id` on the corresponding response event, and a private `_RequestTracker` resolves awaiters by that key. Per-turn tasks are spawned with an empty `contextvars.Context()` so the bus's `_IN_WORKER` ContextVar is reset for the turn тАФ without this the loop's publishes inside a turn would fire-and-forget and correlation futures would never resolve. Guards: `max_iterations` trips `kernel.error` (`source="agent_loop"`, `message="max_iterations_exceeded"`); `user.interrupt` cancels the turn for that session. Depends only on `bus`, `events`, `plugin` тАФ no `cli`, `plugins`, or `core` imports, and no concrete plugin imports.
- **`source="kernel"` is reserved for kernel-internal code.** The plugin registry (issue #13) will enforce that plugin subscriptions cannot claim this source. Until the registry lands, this is an honor-system invariant тАФ the recursion guard in the bus trusts it to short-circuit loops.
- `loop.py` тАФ ``memory.write`` is emitted after ``assistant.message.done`` and awaited to completion before the turn task returns. A slow memory plugin delays the session's next ``user.message.received``. Memory plugins are expected to be fast (sqlite INSERT scale); strategies that need async fire-and-forget persistence should use a skill plugin instead of ``write_memory``.
- `config_store.py` тАФ live SQLite KV store at `${XDG_STATE_HOME:-~/.local/state}/yaya/config.db` (#104). Replaces boot-time TOML/env as the source of truth after a one-time migration stamped with `_meta.migrated_from_toml_at`. Single `sqlite3.Connection` + single-worker `ThreadPoolExecutor` (same pattern as `memory_sqlite`), in-memory cache backs every read via `ConfigView` (read-only `Mapping` proxy, supports scoped views `view(prefix="plugin.<ns>.")`). Every `set` / `unset` / migration write emits `config.updated` on `session_id="kernel"` (lesson #2) with payload `{"key", "prefix_match_hint"}`; plugins filter by `key.startswith("plugin.<name>.")` and rebuild state тАФ `llm_openai` re-instantiates its `AsyncOpenAI` client on `api_key` / `base_url` change, `strategy_react` reads `ctx.config["provider"]` per decision so `yaya config set provider anthropic` flips the next `strategy.decide.response` with no restart. `PluginRegistry` opens the store before plugin discovery and closes it after every `on_unload` so teardown writes still reach the DB; caller-supplied stores are left open.
- `config.py` тАФ `KernelConfig` (pydantic-settings) resolves settings in a fixed merge order: CLI flags тЖТ `YAYA_*` env (with `__` nested delimiter) тЖТ `$XDG_CONFIG_HOME/yaya/config.toml` тЖТ defaults. Custom `_NestedEnvExtras` source captures `YAYA_<NS>__<KEY>` env vars for non-declared namespaces (pydantic only honours `env_nested_delimiter` for declared fields, so plugin sub-trees need this). `KernelConfig.plugin_config(name)` returns the sub-tree fed to `KernelContext.config` by the registry; absent sub-trees return `{}`. No auto-create on first run.
- `logging.py` тАФ `configure_logging(KernelConfig)` wires loguru sinks (stderr at the config level, rotated DEBUG file under `$XDG_STATE_HOME/yaya/logs/yaya.log`, JSON-per-line on `YAYA_LOG_JSON=1`). Idempotent; safe to call from any process entry. Installs a stdlib `logging` intercept handler on the root so third-party libraries still appear in the unified stream тАФ the kernel modules deliberately keep `import logging` + `_logger = logging.getLogger(__name__)` and route through that intercept rather than importing loguru directly. `get_plugin_logger(name)` returns `logger.bind(plugin=name)` so plugins receive a pre-bound logger via `KernelContext.logger` (typed as `Any` to keep loguru out of the Plugin ABI). The redaction filter scrubs any `record["extra"]` field whose key matches `r".*(token|key|secret|password|passphrase).*"` (case-insensitive) and any value shaped like `sk-...` / `Bearer ...`.
- `llm.py` тАФ v1 llm-provider contract. `LLMProvider` Protocol (`name` / `model_name` / `thinking_effort` / async `generate(system_prompt, tools, history) -> StreamedMessage`). `StreamedMessage` Protocol yields `ContentPart | ToolCallPart` + terminal `TokenUsage`. `TokenUsage` pydantic model with Anthropic cache counters (`input_other` + `input_cache_read` + `input_cache_creation` тЖТ derived `input`; + `output` тЖТ `total`); derived values ship via `model_serializer(mode="wrap")` to sidestep mypy/pyright asymmetry on `@computed_field`+`@property`. Closed error taxonomy `ChatProviderError` тЖТ `APIConnectionError` / `APITimeoutError` / `APIStatusError` (carries `status_code` + `request_id`) / `APIEmptyResponseError`, all subclass `YayaError`. Three lazy-import SDK converters: `openai_to_chat_provider_error`, `anthropic_to_chat_provider_error`, `convert_httpx_error`. `RetryableChatProvider` Protocol is a frozen hook shape; runtime lands separately. Mechanical SDK-only enforcement via `scripts/check_banned_frameworks.py::check_llm_plugin_imports` (scoped to `src/yaya/plugins/llm_*/`). Bundled `llm_openai` + `llm_echo` stay on the legacy `on_event` subscribe path in this PR.
- `approval.py` тАФ HITL approval runtime (#28). Per-bus singleton installed by the registry between plugin load and `kernel.ready`. `Tool.pre_approve` default delegates here when `requires_approval=True`; the runtime emits `approval.request`, awaits `approval.response`, and surfaces `approve` / `approve_for_session` / `reject` to the dispatcher. ALL `approval.*` events route on the reserved ``"kernel"`` session (lesson #2) тАФ the originating tool-call session worker is blocked on `await pending_future` from inside the dispatcher, so the response MUST deliver on a different worker to resolve. `approve_for_session` caches `(tool_name, params_fingerprint)` under the tool-call's session id (not the routing session) so future identical calls short-circuit. Timeout (60s default) тЖТ `approval.cancelled` + `ApprovalCancelledError`; explicit reject or timeout тЖТ dispatcher emits `tool.error(kind="rejected")`. `_pending` dict is popped on every exit (success / timeout / shutdown) so long-running processes do not leak futures (lesson #6). Subscription `source="kernel-approval"`.
- `tool.py` тАФ v1 tool contract. `Tool(pydantic.BaseModel)` declares params as fields + `name` / `description` / `requires_approval` ClassVars + `async run(ctx) -> ToolReturnValue`. `ToolOk` / `ToolError` envelope (with `brief тЙд 80 chars` + `DisplayBlock`; built-in `TextBlock` / `MarkdownBlock` / `JsonBlock`). Module-level `_tool_registry` populated by `register_tool()` and drained by `unregister_tool(name)` (idempotent; returns `True` when a row was removed, `False` otherwise тАФ adopted by `mcp_bridge` and `agent_tool` `on_unload` so hot-reload leaves no stale tool classes, #90); `dispatch()` coroutine subscribed to `tool.call.request` via `install_dispatcher(bus)` handles only payloads with `schema_version="v1"` (backward compat for legacy `on_event` tools). Validation happens before the subclass is instantiated; failure emits `tool.error` kind=validation and `run()` is never called. `pre_approve()` hook shape is present; real approval runtime lands in #28. Kernel-origin subscriber (source=`"kernel"`) so bus recursion guard drops rather than re-emits on handler failure.
- `errors.py` тАФ closed taxonomy: `YayaError` (base) тЖТ `KernelError` (crash-worthy), `PluginError` (recoverable; bus-isolated, becomes a `plugin.error` event with `kind` + 8-char `error_hash`), `ConfigError` (user-facing), `YayaTimeoutError` (generic; deliberately NOT shadowing `builtins.TimeoutError`). Plugins own their own subclasses under `PluginError` (e.g. `OpenAIError(PluginError)`).
- `compaction.py` тАФ conversation compaction (#29). `Summarizer` Protocol (`async summarize(entries, target_tokens) -> str`); `estimate_text_tokens` (char/4 heuristic, tokenizer-agnostic; deterministic per AC-03); `should_auto_compact(tokens, *, threshold)` (pure gate, `threshold<=0` disables); `compact_session(session, summarizer, *, target_tokens, bus)` drains post-last-anchor entries, runs the summariser, appends an anchor with `state={"kind": "compaction", "summary": ..., "tokens_before": ...}`, emits `session.compaction.{started,completed,failed}` on `session_id="kernel"` (lesson #2 тАФ the originating session worker is still draining the turn that triggered the check). `CompactionManager` subscribes to public kinds, schedules compaction on threshold breach, guards single in-flight per session via `OrderedDict` keyed by `session_id`, retries up to 3 times with exponential backoff, FIFO-evicts the in-flight/attempts/disabled maps at `_INFLIGHT_CAP=1024` (lesson #6). Per-attempt failures are suppressed (`compact_session(..., _emit_failure=False)`); a **single terminal** `session.compaction.failed` event with `attempts: int` lands after the retry budget exhausts тАФ adapters see one failure per retry chain. `LLMSummarizer(provider)` bridges the `Summarizer` Protocol to the v1 `llm-provider` contract тАФ `yaya serve` wires it onto the first loaded `llm-provider` plugin when `compaction.auto=true`, warning-and-skipping when no provider is available (lesson #23). `Session.compact()` is a thin lazy-import wrapper. The default `select_messages` context selector detects compaction anchors (`payload.state.kind == "compaction"`) and injects the stored summary as a `role="system"` message **while eliding every message accumulated before the anchor** тАФ the summary replaces the pre-anchor prefix. `SessionPersister._SKIP_KINDS` excludes the three compaction events so a future wiring mistake still does not land them on a user tape.
- `session_context.py` тАФ multi-connection runtime wrapper over `Session` (#36). `Connection` is the per-client handle (uuid + `send_cb` + heartbeat). `SessionContext` holds a bounded connection registry (default cap 64, lesson #6) plus a cancellable heartbeat reap task (default 60s). `fanout(event)` pushes to every attached connection; send failures translate to a quiet `detach(reason="send_failed")` (lesson #29). Reconnect replay: `attach(..., since_entry=N)` takes a per-connection `asyncio.Lock`, emits one `session.replay.entry` per tape entry with `entry.id > N`, and closes with `session.replay.done` тАФ live events arriving mid-replay buffer behind the lock so nothing is dropped or duplicated. All `session.context.*` / `session.replay.*` envelopes route on `session_id="kernel"` (lesson #2). `SessionManager` owns one context per live `session_id` and routes bus events via `kinds` subscriptions. `install_session_manager()` mirrors the persister install helper. Depends only on `bus`, `events`, `errors`, `session` тАФ no plugin / CLI imports.
- `registry.py` тАФ `PluginRegistry` discovers plugins via `importlib.metadata.entry_points(group="yaya.plugins.v1")`, drives `on_load` / `on_event` / `on_unload`, and isolates repeat offenders via a consecutive-`plugin.error` counter (threshold default 3). `PluginRegistry.loaded_plugins(category)` returns live plugin instances filtered by category тАФ the public lookup `yaya serve` uses to pick an `llm-provider` for the compaction summariser (first loaded wins; warns when >1). Bundled vs third-party is a deterministic load-order tie-breaker only тАФ both go through the **same** `_load_entry_point` code path. The failure-accounting handler subscribes with `source="kernel-registry"` (NOT `"kernel"`, which trips the bus's recursion guard); when the threshold trips, the unload task is spawned via `asyncio.create_task(..., context=contextvars.Context())` so the bus's `_IN_WORKER` ContextVar resets and the task's `plugin.removed` publish is observed by adapters тАФ same pattern `AgentLoop._on_user_message` uses. `install` / `remove` shell to `uv pip` via `asyncio.create_subprocess_exec` (never `shell=True`); `remove` rejects bundled plugins with `ValueError`; `validate_install_source` rejects shell metachars, unsupported URL schemes, and empty input before any subprocess spawns. `_PluginRecord` is `@dataclass(eq=False, slots=True)` so subscription handles keep identity semantics (lesson #7). Stdlib only; depends on `bus`, `events`, `plugin` тАФ no imports from `cli`, `plugins`, `core`, or `loop`.
- Stdlib only (plus `loguru` only if a plugin-facing logger demands it). No module-level side effects at import.

## Interaction (patterns)
- New public event kind тЗТ update `PublicEventKind`, the per-kind `TypedDict`, [`docs/dev/plugin-protocol.md`](../../../docs/dev/plugin-protocol.md), and `GOAL.md`'s category table in the SAME PR. Label `governance`.
- Do NOT add wildcard subscription at 1.0 тАФ kind match is exact. Adapters that want many kinds subscribe many times.
- Do NOT let plugins emit `plugin.error` or `kernel.error` directly. Only the kernel synthesizes them.
- Do NOT import from `cli`, `plugins`, or `core`. Direction is one-way: everything depends on `kernel`, never the reverse.
- Do NOT add a "fast path" for bundled plugins. They use the same ABI.
